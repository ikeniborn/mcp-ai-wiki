Требования по разделам

РазделТребование
§Problem · R1Добавить MCP-тул, который правит один ## раздел существующей страницы, переиндексирует только затронутый раздел, коммитит и пушит в git. Сегодня править контент можно лишь ручной правкой .md + wiki_index или delete-recreate.
§Problem · R2Любая мутация вики коммитится и пушится. Сейчас write_page/create_domain коммитят локально; wiki_index не коммитит; push только через wiki_sync.
§Key existing behaviorПроверенные факты о коде (не переизобретать): инкрементальный эмбеддинг (index_domain переэмбедит только изменённые чанки); chunk режет по ##, id = file#heading, ## Overview исключён из индекса и префиксится в каждый чанк; _stale берёт первую лог-запись по странице; git fail-soft.
§ApproachВыбран in-place section splice. Отклонены: delete + recreate (двухфазно, не атомарно, теряет историю лога); overwrite всей страницы (splice ложится на caller).
§Components/1 · wiki_update_pagewiki_update_page(domain, slug, heading, new_body, source=None); heading — текст без ##. Транзакция с откатом: read original → replace_section → validate_page (блок при deep_heading/pre_h2_text) → write → upsert лога (если source) → index_domain → commit_and_push. Откат: восстановить файл и байты лога. Возврат: page/heading/indexed_chunks/reused/embedded/bytes/over_cap/committed/pushed.
§Components/2 · engine/section.pyНовый stdlib-модуль: replace_section(content, heading, new_body) + SectionError. Тот же _H2 regex (keep-in-sync). Матч по heading.strip(); not-found и duplicate → SectionError. Правка ## Overview перечанкует всю статью (допустимо).
§Components/3 · ingest-log upsertПри source — переписать log.jsonl, убрав прежние ingest-записи страницы, добавив свежую с новым src_hash (иначе _stale first-hit-wins проигнорирует). Rollback caveat: не переиспользовать _rollback_last_ingest_log (только последняя строка) — нужен полный snapshot байтов лога.
§Components/4 · commit_and_pushcommit_and_push(base, message, pathspec) = auto_commit → если committed, sync(base). Маппинг: не committed → {committed:False, pushed:False}, sync не вызывается; committed → pushed = sync().get("pushed", False), warning прокинут. Ретрофит: write_page, update_page, create_domain, index — все с pathspec=<domain>. wiki_index теперь коммитит+пушит.
§Error handlingВсё fail-soft через @_safe: page-missing, heading not-found/duplicate, block-структура, source в .iwikiignore, сбой write/log/index (откат + raise), push-fail (pushed:false + warning).
§Testingengine/section.py: replace ok/not-found/duplicate/Overview. wiki_update_page: правка+reindex+upsert; page-not-found; block-heading reject; откат при сбое index; .iwikiignore. commit_and_push: git-repo → committed+sync invoked; non-repo fail-soft. Регресс: write/create_domain/index зелёные.
§Out of scopeНет section-scoped single-file индекса (index_domain перечитывает домен — дёшево); нет режима «создать раздел, если нет» (update требует существующий).
§Follow-up upkeepREADME.md (тул в список), docs/wiki/architecture.md (транзакция update + commit-and-push), iwiki-домен iwiki-mcp, patch-bump версии.
